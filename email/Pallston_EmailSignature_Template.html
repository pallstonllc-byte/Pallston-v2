About the logo & "no attachment / no broken image" goal:

The <img src="https://www.pallston.com/..."> tag above points to a logo file hosted on a stable, public HTTPS URL. This is the approach every major signature platform (HubSpot, Exclaimer, WiseStamp, etc.) uses, for a specific reason: it's the only method that is consistently attachment-free and full-quality across Gmail, Outlook (web, desktop, new), and Apple Mail at the same time.

What to do: Host pallston-logo-email-signature.png (342×88px, transparent background, already 2x-resolution for the 44px display height used here) at a permanent, never-changing URL on your own domain or CDN — then swap the src above. width/height are already set to the correct 171×44 display size so it renders crisp on retina screens and the layout doesn't jump while images load.

Why not just paste the image into Outlook's signature box? That's almost certainly what's causing logos to show up as an attachment today — pasting an image directly embeds it as a CID inline attachment, which some recipients' clients render as a literal attachment icon.

Why not embed as base64 instead? A base64 data-URI image (the logo's bytes encoded directly into the HTML) does avoid hosting — but classic Outlook for Windows has unreliable support for base64 images in signatures specifically (it can fail to render or strip it on send), and it adds the full image size to every single email sent, which can trip spam filters at scale. For one-off use it's fine; for a company-wide standard, hosted URL is the safer bet. If you'd rather go this route anyway, upload your real logo file here and I'll generate the base64 version for you.

On Aptos: Aptos currently only ships natively with Microsoft 365 apps. Recipients on Gmail, Apple Mail, or older Outlook won't have it installed, so the signature (and ideally the email body too) should use a font stack like Aptos, Calibri, Arial, sans-serif — that way it renders as Aptos wherever available and falls back gracefully elsewhere instead of defaulting to Times New Roman.
